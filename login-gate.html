Client-side Login Bypass

Category: Auth · Technique: Read the login JavaScript
Brief

A lazy developer wrote a "secure" login page — except the whole check runs in the browser. The correct credentials are written right into the JavaScript source. Anyone can read them.

This is one of the most famous anti-patterns in web security: client-side authentication. Authentication must happen on a server you control. Putting it in the browser means the user is also the guard.

Hint View the page source (or open DevTools → Sources) and read the <script> at the bottom. The username/password comparison is right there in plain text.

Real-world takeaway

Anything your browser can check, your browser can skip. Password comparisons, role checks, "premium unlocked" flags — if the logic runs on the client, the client controls it. Real auth sends credentials to a server that holds the ground truth in a database the user never sees.
