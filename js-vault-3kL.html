Base64 JavaScript

Category: Decoding · Technique: Read inline JS + decode Base64
Brief

Developers love to "hide" strings by encoding them in Base64. It's trivially reversible — anyone with a browser console can decode it in one line. Never rely on Base64 for security. It's obfuscation, not encryption.

Two variables are waiting for you at the bottom of this page's inline <script>. Decode them and you've got the flag and the next path.

Hint The script on this page declares variables that look like gibberish but follow a familiar alphabet — letters, digits, a few symbols, the trailing equals sign. The browser already ships with a function that will turn that gibberish back into text. Find it in the console docs.
What the script holds

Two declarations sit at the bottom of this document. Their values are strings — not random strings, but the kind a printable encoding produces. Both decode cleanly. One is the flag; the other is the next path. Which is which is up to you to read.

The encoding is one of the oldest in the web stack. It has been sitting in window since browsers existed.
