Directory Fuzzing

Category: Recon · Technique: Path discovery
Brief

Some directories are not linked anywhere on the site. They exist, but no page navigates to them, and they don't show up in robots.txt. They're hidden purely by being unguessable — which is why hunters guess.

The technique is called directory fuzzing — you try a wordlist of common names against the server and watch which ones answer with content instead of a 404. Build, backup, staging, internal tools — admins reach for the same handful of folder names every time.

Hint You may already have seen a clue to this folder's name on a previous page — admins like to list what they want hidden in one convenient file. Think about what an internal team would call a folder that nobody outside engineering should reach. Compose a few candidates by hand — the URL bar is your fuzzer.
Why it works

Web servers often serve a directory's index.html automatically when you visit the folder path. So /secret/ will serve /secret/index.html without showing it in the URL.

Real hunters automate this with tools like ffuf, gobuster, or feroxbuster against a wordlist. Today, you can do it by hand — there's only a handful of candidates.

The folder you want serves an index.html — so once you guess right, you'll know.
