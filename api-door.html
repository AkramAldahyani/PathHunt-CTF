Hidden API

Category: Network · Technique: Inspect XHR / fetch calls
Brief

The page is quiet, but the network is chatty. Modern websites pull data in the background from API endpoints — JSON feeds, config files, analytics pings. Each of those requests is visible in DevTools, and each response is a potential leak.

This page is silently calling an internal endpoint right now. Find it, read the response, and you've got the flag.

Hint Open DevTools → Network tab → filter by Fetch/XHR → reload the page. Click the request that isn't a CSS, JS, or HTML file. Check its Response or Preview tab.

Why APIs leak

API developers often assume "nobody will call this endpoint directly" — so they skip authentication. The front-end page calls it politely, displays only a slice of the data, and everyone's happy. Until a hunter opens DevTools.

Common real-world findings:

  • Full user records returned when the UI only shows a name
  • Debug endpoints like /api/debug/config
  • API keys embedded in client-side JS
