HTTP Headers

Category: Protocol · Technique: Header inspection
Brief

Every HTTP response carries headers — invisible key/value pairs that travel alongside the HTML. Servers use them to send instructions, version info, and sometimes things they didn't mean to expose.

This page has a custom X-Flag header attached to its response. It never appears in the rendered page — you have to inspect the HTTP response directly.

How to find it
  • curl: curl -I http://localhost:3000/header-secret.html — requires npx serve .
  • DevTools Network tab: F12 → Network → reload → click header-secret.html → Response Headers

Start the server with npx serve . from the project root — it reads serve.json and injects the custom headers automatically.

Response headers (partial)

Standard headers from the server. The custom x-* headers are not shown here — check the page source.

Status: 200 OK
Content-Type: text/html; charset=utf-8
Server: pathhunt-edge/1.4
Cache-Control: no-store
X-Powered-By: curiosity
X-Flag: [inspect the page source]
Why it matters

Real-world sites routinely leak information through headers: framework versions (X-Powered-By: Express), internal hostnames, debug tokens, even stack traces on error pages. Reading headers is always step one of recon.

On a real target: curl -I https://example.com prints only response headers. curl -v shows everything including request headers.
