HTTP Headers

Category: Protocol · Technique: Header inspection
Brief

Some secrets travel above the page. Every HTTP response carries headers — key/value pairs the server sends along with the HTML. They're invisible in the rendered page, but your browser reads them and so can you.

Since this is a static hunt (no real server), the same headers are mirrored into the page's <meta> tags in the <head>. Same game: inspect what the document declares about itself.

Hint Open DevTools → Elements tab → find the <head> section. Look for <meta name="x-..."> tags. On a real server, you'd look in the Network tab → click the request → read the Response Headers panel.
Response headers (simulated)
Status: 200 OK
Content-Type: text/html; charset=utf-8
Server: pathhunt-edge/1.4
Cache-Control: no-store
X-Powered-By: curiosity
X-Flag: flag{header_hunter}
X-Next: /cookie-room.html
Why it matters

Real-world sites leak plenty through headers: framework versions (X-Powered-By: Express), debug tokens, internal hostnames, even stack traces on error pages. Always read the headers.

Also: curl -I https://example.com prints just the headers from the command line.
