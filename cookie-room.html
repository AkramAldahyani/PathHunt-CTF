Cookie Tampering

Category: Client trust · Technique: Cookie modification
Brief

Cookies are key/value pairs stored by your browser and sent with every request. Some sites use cookies to track what role the user has — role=user, role=admin, and so on.

The mistake: trusting the browser to tell the truth. If the server never verifies the role on its end, you can simply change the cookie and promote yourself. Real sites have fallen to this exact bug.

Hint The page sets a cookie that holds the answer to "who are you?" — and then asks the browser the same question back. Whatever the cookie says is taken as truth. The browser is yours; the cookie store is yours. Pick a value the page would consider privileged.

Real-world note

Well-designed sites never put trust decisions in a plain cookie. Instead they use signed sessions (JWT, encrypted cookies) so tampering is detected and rejected server-side. Any decision about "is this user an admin?" lives on the server, not in a cookie the user can edit.
